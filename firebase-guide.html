🔥 Firebase Setup Guide

For your Mental Wellness Tracker — beginner friendly!

📦 Your Firestore Database Schema

Think of Firestore like a set of folders (called collections) that hold files (called documents). Here's what your database will look like:

📁 Collection: moods — used by index.html
📄 Document ID: auto-generated by Firebase
• mood string — e.g. "😊 Happy"
• time timestamp — when it was saved
📁 Collection: calendar — used by calander.html
📄 Document ID: the date e.g. 2026-4-24
• mood string — "happy", "neutral", or "sad"
• journal string — the note written for that day
📁 Collection: journals — used by journal.html
📄 Document ID: the date e.g. 2026-04-24
• text string — the journal entry
• date string — e.g. "2026-04-24"
• savedAt string — exact time it was saved
💡 You don't need to create these collections manually! Firebase creates them automatically the first time you save data.

🚀 Step-by-Step Setup

1Go to Firebase Console

Open your browser and visit: console.firebase.google.com

Sign in with the Google account you used to create the project.

2Open your project

Click on "mental-wellness-tracker-b85e4" — this is the project already set up in your code.

3Enable Firestore Database

In the left sidebar, click Build → Firestore Database.

If it asks you to create a database, click "Create database".

When asked about rules, choose "Start in test mode" (this lets you read/write freely while developing).

Choose a location (any is fine, e.g. us-central) and click Enable.

⚠️ Test mode expires after 30 days. For a real project, you'll need to update security rules later. For now, test mode is fine.
4Replace your files

Replace your existing calander.html and journal.html with the updated files provided. Your index.html already had Firebase — no changes needed there.

5Test it!

Open calander.html in your browser → click a date → set a mood → click Save.

Then go to Firebase Console → Firestore Database → you should see a calendar collection appear with your data! 🎉

✅ How to Verify It's Working

In Firebase Console → Firestore Database:

After saving a mood in calander.html, you should see:
  📁 calendar → 📄 2026-4-24 → mood: "happy"

After saving a journal in journal.html, you should see:
  📁 journals → 📄 2026-04-24 → text: "..."

🐛 Common Errors & Fixes

Error: "Permission denied"

Go to Firebase Console → Firestore → Rules tab → make sure the rules say:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;  // Test mode - allows everything
    }
  }
}
Error: "Firebase App named '[DEFAULT]' already exists"

This happens if Firebase is initialized twice on the same page. Each page should only initialize Firebase once — the updated files already handle this correctly.

Data not showing after refresh?

Check the browser Console (press F12 → Console tab) for any red error messages and note what they say.
